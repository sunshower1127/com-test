# 롤백(세이브포인트) 구현 상태 점검

## 상태: 🔲 미착수

## 목표

LLM 코드 실행 실패 시 문서를 자동 복원하는 세이브포인트 기능이
실제로 동작하는지 점검하고 보완.

## 확인 항목

- [ ] executor.ts의 `executeWithSavepoint` 현재 구현 확인 — 실제 저장/복원인지 스텁인지
- [ ] Excel 세이브포인트: `SaveCopyAs` → 임시 파일 저장 → 에러 시 `Close(false)` + `Open(tempPath)` 동작 확인
- [ ] HWP 세이브포인트: `SaveAs` 또는 `Save` → 임시 파일 → 에러 시 `Clear(1)` + `Open(tempPath)` 동작 확인
- [ ] 새 문서(아직 저장 안 된)일 때의 처리 — 세이브포인트 어떻게?
- [ ] 임시 파일 경로 관리 (os.tmpdir, 정리 타이밍)
- [ ] 에러 시 `restored: true/false` 플래그가 정확히 반환되는지
- [ ] 다중 앱 동시 실행 시 (Excel + HWP) 각각 독립 세이브포인트

## 설계 원칙 (기존 합의)

- 세이브포인트는 인프라 레벨에서 자동 처리
- LLM은 롤백 존재를 모름 — 비즈니스 로직만 생성
- 실행 전 자동 저장, 실패 시 자동 복원, 성공 시 임시파일 삭제
