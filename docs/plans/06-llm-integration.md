# Milestone 6: LLM 연동

## 목표

LLM이 사용자의 자연어 요청을 JS 코드로 변환하고,
에러 시 자동 재시도하여 오피스 문서를 조작.

## 핵심 작업

### 6-1. 프롬프트 설계

```
시스템 프롬프트:
- 역할: 오피스 문서 자동화 코드 생성기
- 사용 가능 객체: excel, hwp (Proxy 기반 COM 객체)
- COM 문법 설명 + 제약사항
- Cheat Sheet 주입 (Milestone 2 산출물)

예시:
  User: "A1~A10에 1부터 10까지 넣어줘"
  Assistant:
    for (let i = 1; i <= 10; i++) {
      excel.ActiveSheet.Cells(i, 1).Value = i;
    }
```

### 6-2. Cheat Sheet 주입

Milestone 2에서 생성한 앱별 cheat sheet를 시스템 프롬프트에 포함:
- 현재 활성 앱에 해당하는 cheat sheet만 주입 (토큰 절약)
- 주요 객체의 프로퍼티/메서드 요약

### 6-3. 에러 재시도 루프

```
최대 3회 시도:

1차: 세이브포인트 생성 → LLM 코드 생성 → 실행
  └── 성공 → 세이브포인트 삭제, 결과 반환
  └── 실패 → 세이브포인트 복원, 에러 정보 수집

2차: 원본 코드 + 에러 메시지 + 스택트레이스 → LLM에 재전송
  (문서는 이미 세이브포인트로 복원된 상태)
  "이 코드에서 에러가 발생했습니다:
   [에러 메시지]
   [해당 라인]
   문서는 실행 전 상태로 복원되었습니다.
   수정해서 다시 작성해주세요."
  └── 성공 → 결과 반환
  └── 실패 → 세이브포인트 복원, 3차 시도

3차: 누적 에러 정보 포함 → 최종 시도
  └── 실패 → 사용자에게 에러 보고
```

### 6-4. 안전장치 (연구 기반)

Self-Refine 논문 등에서 확인된 best practice:

| 안전장치 | 설명 |
|---------|------|
| 재시도 상한 | 최대 3회 (2~3회에서 대부분의 가치 발생) |
| 동일 에러 감지 | 같은 에러 2회 연속 → 즉시 중단 |
| 동일 코드 감지 | LLM이 같은 코드 재제출 → 즉시 중단 |
| 토큰 예산 | 전체 retry 루프의 토큰 상한 설정 |
| 세이브포인트 복원 | 매 실패 시 문서 자동 롤백 (Milestone 4) |

### 6-4. 실행 결과 피드백

LLM에게 실행 결과도 돌려줘서 대화 이어가기:
```
User: "A열 데이터 합계 구해줘"
LLM: [코드 생성 + 실행]
System: "실행 결과: result = 1550"
LLM: "A열의 합계는 1,550입니다."
```

### 6-5. LLM API 선택

- Claude API (Anthropic) — tool use 활용 가능
- 코드 생성 부분을 tool로 정의하면 구조화된 응답 가능
- 또는 단순 text completion으로 코드 블록 추출

## 검증

1. "A1에 hello 넣어줘" → 코드 생성 → 실행 → Excel에 반영
2. 의도적 에러 유발 → 재시도 → 자동 수정 확인
3. 복잡한 요청: "A열에서 중복 제거하고 B열에 정렬해서 넣어줘" → 다단계 코드
